本地环境搭建

Hexo是基于 Node.js 开发的，因此需要先安装Node.js。Node.js有“长期支持版”和“当前发布版”两个版本，当前发布版版本较新，相应配套组件更新不及时，后续安装使用Hexo会出现一些未知错误（比如我曾将遇到过的一个bug），因此注意下载使用长期支持版。Node.js 软件的安装过程中，除了安装路径外，其它都采用默认方式。

初始化指令会自动创建一个名为 IchBlog 的文件夹并下载初始化文件，不需要额外手动额外创建。 安装完成后会有一个 ‘Start blogging with Hexo!’ 的提示。这个步骤会遇到的一个 bug 是“无法加载文件脚本”的问题：

解决思路来自 webpack : 无法加载文件。首先以管理员身份打开Windows PowerShell，输入get-ExecutionPolicy，返回显示 Restricted（表示状态是禁止的），然后输入set-ExecutionPolicy RemoteSigned更改策略，输入A选择全是，再次输入get-ExecutionPolicy可以看到已经更改成功，再次执行hexo命令即可正常执行。

服务器：GitHub 个人仓库

总体思路为依托 GitHub 个人仓库存储网站。首先注册一个 GitHub 账号，记住注册时的用户名，邮箱和密码。然后创建一个名为 “xxx.github.io” 格式的公开个人仓库，其中 “xxx” 替换为 GitHub 账号的用户名即可。回到电脑端，安装 Git 软件。Git 是分布式版本控制系统，通过ssh公钥连接后就可以实现 Hexo 代码在本地电脑和服务器之间同步。安装完成后打开 “Git Bash”，输入自己在GitHub上注册的用户名和邮箱：

然后输入生成ssh公钥的指令：

在生成公钥过程中会弹出一些选项，不需要更改，一直按回车即可。之后就可以在公钥生成过程中提示的文件路径（C/Users/XXX/）下找到一个“.ssh”的文件夹，文件夹中有“id_rsa”和“id_rsa.pub”两个文件，其中“id_rsa.pub”就是用于连接本地和云服务器的公钥，相当于本地电脑有了一把访问服务器的“钥匙”，后续工作需要把这个公钥配置到云服务器上。
在 GitHub 网站中，依次进入 “Settings” -> “SSH and GPG keys”, 在 “SSH keys” 一栏点击 “New SSH key”。 在电脑上用文本编辑器打开上一步创建的 “id_rsa.pub”，把里面的内容复制粘贴到 GitHub 网站 “New SSH key” 中， “Title” 内容可编辑为个人博客名称，方便识别。
